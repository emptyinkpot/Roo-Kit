# 贡献规范

## 这个仓库适合的改动

- Roo / Cursor 的 skill、MCP、治理规则与运行时配置
- 编排、权限、sandbox、能力注册、能力说明文档
- 作为“外置工程助手层”的基础设施资产

## 默认不应混入的改动

- 业务仓长期实现代码
- 无任务指向时对扩展打包产物、数据库、缓存、临时状态的顺手修改
- 为兼容旧路径而长期保留的平行旧/新实现

## Reviewer / Sourcery 会重点看什么

- 是否保持 env-first、声明驱动、最小权限原则
- 是否引入重复的 skill / MCP / capability 根
- 是否把业务实现错误地迁进 Roo 目录
- 路径与权限变更是否附带了最小可复现实证

## PR 期望内容

每个 PR 请说明：

1. 本轮修改的是哪类资产（skill / MCP / 配置 / 文档）
2. 影响了哪些路径解析、权限或运行时行为
3. 如何验证
4. 是否需要同步相关 README 或治理文档

## 跳过 Sourcery 的规则

默认不要跳过。如果确实需要跳过，请在 PR 上加：

- `skip-sourcery`
- `no-sourcery`
- `sourcery-ignore`

并在 PR 描述里说明原因。
